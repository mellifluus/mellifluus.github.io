2. Algorithms Analysis Theory

In this section, I’m going to give an overview on the practices used to analyze algorithms. It’s imprecise to just measure how much time or space an algorithms requires to run to determine its complexity. That would just give an information on how much resources the algorithm takes to execute on one specific machine, but doesn’t really give unbiased information on what’s the amount of resources required on a worse or even better machine. That is why, while analyzing algorithms, we use specific notation that allows for clearer understanding of the complexity of an algorithm.

2.1. Complexities

There are two types of complexities to take into consideration when designing an algorithm:

  • Time complexity – Attempts to give a broad idea of how complex a given algorithm is in correlation with its execution time.
  • Space complexity – Attempts to give a broad idea of how complex a given algorithm is in correlation with the space required for its execution. It consists of two parts, which are summed together:
    • Auxiliary space – Represents the amount of space used by variables and constants that are not dependent by the size of the input of the algorithm. In other words, the temporary space required.
    • Variable space – Represents the amount of space used by the input size of the algorithm.

Both complexities can be expressed using Asymptotic Notation, which is going to be covered in depth in the next chapter. In this document, I’m going to be – almost – solely focus on the concept of time complexity.

2.2. Asymptotic Bounds

Any given algorithm takes an n amount of steps to complete. As stated before, we can’t just measure the time a step takes to complete as that wouldn’t give us a realistic complexity representation. That is why, to express the complexity of an algorithm we make use of a concept called Asymptotic Notation, which allows for a machine-independent measurement.

The notation makes use of the concept of Asymptotic Bound which is, by definition, a curve representing the limit of a function where the distance between the function and the curve tends to zero.

When analyzing an algorithms, we’re interested in three types of bounds (each with its own notation).

2.2.1. Asymptotic Upper Bound – \(Big\mbox{-}O\)

The upper bound, which can also be referred to as \(Big\mbox{-}O\) or just \(O\) represents the worst-case scenario in the running time of an algorithm. Let’s say \(f(n)\) describes the running time of an algorithm; we can then say that \(f(n)\) is \(O(g(n))\) if and only if a constant \(c\) exists such that \(c \cdot g(n) \geq f(n)\) for all \(n \geq n_0\).

In mathematical terms, we can then define \(Big\mbox{-}O\) as follows:

\(f(n) = O(g(n)) \iff c \cdot g(n) \geq f(n) \;\; \forall \;\; n \geq n_0 \)

More than often, there is more than one function that is suitable as \(Big\mbox{-}O\) for one algorithm, but we’re interested in the tightest (closer) upper bounding function among them, since it gives us a more realistic idea of the worst-case running time of a function.

fig. 1

2.2.2. Asymptotic Lower Bound – \(Big\mbox{-}\Omega\)

The lower bound, which can also be referred to as \(Big\mbox{-}\Omega\) or just \(\Omega\) represents the best-case scenario in the running time of an algorithm. Let’s say \(f(n)\) describes the running time of an algorithm; we can then say that \(f(n)\) is \(\Omega(g(n))\) if and only if a constant \(c\) exists such that \(c \cdot g(n) \leq f(n)\) for all \(n \geq n_0\).

In mathematical terms, we can then define \(Big\mbox{-}\Omega\) as follows:

\(f(n) = \Omega(g(n)) \iff c \cdot g(n) \leq f(n) \;\; \forall \;\; n \geq n_0 \)

Again, there is usually more than one function that is suitable as \(Big\mbox{-}\Omega\) for one algorithm, but we’re interested in the tightest (closer) lower bounding function among them, since it gives us a more realistic idea of the best-case running time of a function.

fig. 2

2.2.3. Asymptotic Tight Bound – \(Big\mbox{-}\Theta\)

The tight bound, which can also be referred to as \(Big\mbox{-}\Theta\) or just \(\Theta\) represents the average-case scenario in the running time of an algorithm. Let’s say \(f(n)\) describes the running time of an algorithm; we can then say that \(f(n)\) is \(\Theta(g(n))\) if and only if two constants \(c_1, c_2\) exist such that \(c_1 \cdot g(n) \leq f(n) \leq c_2 \cdot g(n)\) for all \(n \geq n_0\).

In mathematical terms, we can then define \(Big\mbox{-}\Theta\) as follows:

\(f(n) = \Theta(g(n)) \iff c_1 \cdot g(n) \leq f(n) \leq c_2 \cdot g(n) \;\; \forall \;\; n \geq n_0\)

Naively, the asymptotic tight bound is a function which, when multiplied by two different constants, manages to “sandwich” \(f(n)\).

fig. 3

3. Analysis of Data Structures

Data structures are the foundation of most algorithms. They provide a way to store, organize and retrieve data, and each of them has different functionalities, advantages and disadvantages. Knowing how common data structures work and how complex it is to do various operations on them is extremely important for algorithm design – thus I'm going to cover them before the Algorithm Analysis section.

Data structures can be classified in primitive and non-primitive structures:

  • Primitive Data Structures – They're the building block on which non-primitive data structures a built on. They hold one value (i.e. Integer, Float, Boolean...) in a specific location in memory.
  • Non-primitive Data Structures – They're based on primitive data structures, and can be seen as collections of them. They classify in contiguous and linked structures:
    • Contiguous Data Structures – They store data in contiguous (adjacent) locations in memory. The most known example is the Array data structure.
    • Linked Data Structures – They store data in random locations in memory. Each memory cell stores a value and one or more memory pointers to other elements. The most known example is the Linked List data structure.

3.1. Array

Arrays are contiguous data structures, meaning that the data they hold is located in contiguous memory locations. This makes accessing the elements in the array efficient, since each one of them is assigned an index (fig.5). Two dimensional arrays (fig. 6) can be used to represent matrixes in code by having an array which has other arrays as its elements.

Arrays have different properties depending on the programming language implementation. In C, for example, arrays have a fixed size and only allow elements of one data type. In Javascript instead, their size is dynamic (their size doubles every time they're full) and the developer is allowed to store whatever type they prefer into them.

fig. 6 – 2D Array

Arrays are an extremely common data structure, and other structures, that we're going to look into later are based on them.
Space complexity-wise, they're extremely efficient since they contain only data at every index.

3.2. Linked List

Linked Lists are non-linear data structures which means that, even though they're a collection of data, they don't store it in contiguous memory locations. They instead make use of pointers to keep track of every element in the list, which are themselves stored as part of each element.
The first element of the list is the "HEAD" of the list, and that's the only element that the structure itself needs to keep a reference of, since all subsequent elements are keeping references of each other.

Depending on the use case, there is a couple ways to structure a linked list to best suit the needs of the developer:

  • Singly linked list (fig. 7) – It's a linked list where every node points at the next one, creating a unidirectional chain.
  • Doubly linked list (fig. 8) – It's a linked list where every node points at the previous and at the next one, creating a bidirectional chain.
  • Circular linked list – It's a linked list where its nodes create a closed loop, it can be singly or doubly linked:
    • Circular singly linked list (fig. 9) – It's a singly linked list where the last node points back at the first node (HEAD node)
    • Circular doubly linked list (fig. 10) – It's a doubly linked list where the last node points back at the first node (HEAD node), and the first node points to the last node in the chain.

3.4. Queue

      

        

          
A queue can be visualized like a line of people waiting for their turn at the post office. More people can join the queue by standing at the end of the line, but they'll have to wait for their turn because the post office only serves the person in front of the line, one at a time. In computer science, this principle is called F.I.F.O. (First In, First Out).

          
Queues can be implemented using either an array or a linked list and, in general, the latter implementation's best advantage is its dynamic size. In addition to that, the linked list implementation doesn't run into the problem of running out of space after consecutive successful dequeues – something that gets addressed in array implementations using circular queues.

3.5. Tree

      

        

          
Trees are non-linear data structures that consist of nodes and edges. Every node consists of data and pointers to other nodes. The top node is also called root node and has no parent node. A node is a leaf or external node when it doesn't point to any other node – they're usually located towards the bottom of the tree if the tree is balanced. Edges are basically the connections between nodes, and there is a maximum of \(n - 1\) edges in a tree that has \(n\) nodes.

          
Since trees can be implemented in many different ways, there are some properties that describe them. I'll take into consideration fig. 13 while defining them: 

          
    
            
  • Degree: The maximum number of children nodes allowed for every node. In this case, \(3\).
    • 
            
  • Depth: The number of edges between a node and the root node. In this case, depth is \(0\) for the root node, \(2\) for the rightmost node and \(3\) for the leftmost one.
    • 
            
  • Height: Starting from \(0\) for the leaf nodes, it defines the distance between external nodes and the root node. The root node, in this case, has a height of \(3\) so the whole tree also has a height of \(3\).
    • 
            
  • Level: Starting from \(1\) for the root node, it defines the distance between a node and the root node. In this case, level is \(1\) for the root node, \(3\) for the rightmost node and \(4\) for the leftmost one.
    • 
          

          
As stated before, there are many ways of implementing a tree. In a Skewed binary tree for example, every node has only either a left or right child node, which leads to a structure that is more similar to a linked list.
This document is going to cover only trees of degree \(2\).

3.5.2. Red-Black Binary Tree

      

        

          
Red-black binary trees add a new "color" property to their nodes, which can be either red or black. They implement some more rules on the nodes behaviour based on their color:

          
    
            
  • The root of the tree is a black node.
    • 
            
  • A node cannot have a parent or children node of its same color, except when its child is a leaf node.
    • 
            
  • Leaf nodes are considered black nodes and have a value of null.
    • 
            
  • The path from any node in the tree to any of its leaf descendants contains the same number of black nodes.
    • 
          

          
By implementing these rules, the tree balances itself when inserting or removing nodes.

3.5.3. AVL Binary Tree

      

        

          
An AVL tree (named after its inventors – Adelson-Velsky,Landis) is a self-balancing tree that restores balance if needed after every insertion or deletion. It does that by checking that the height of every node's subtrees does not exceed a threshold, which is usually \(1\). In mathematical terms, an AVL tree has to satisfy \(|B(N)| \leq 1\) for every given node \(N\), where \(B(N) = h_l - h_r\) with \(h_l, h_r\) being the height of the left and right child subtrees respectively. 

          
When \(B(N) > 1\), the tree is left-heavy, and requires either a right rotation or a left-right rotation.

          
When \(B(N) < -1\), the tree is right-heavy, and requires either a left rotation or a right-left rotation.

3.6. Heap

      

        

          
Heaps are complete trees. That means that a heap with \(n\) nodes always has a \(O(log(n))\) height. 

          
    
            
  • Min Heap: It's a tree where every node holds a value that is less or equal than its children's held values. The smallest value in the min heap is located at the root of the tree.
    • 
            
  • Max Heap: It's a tree where every node holds a value that is greater or equal than its children's held values. The largest value in the max heap is located at the root of the tree.
